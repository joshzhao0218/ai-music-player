# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-03-24

### Added
- Complete AI-powered music player application
- Natural language command processing with Qwen AI
- Mood detection (relaxing, happy, energetic, sad)
- Genre recognition (Jazz, Rock, Pop, Classical, Electronic, Lofi)
- Full playback control (play, pause, resume, stop, next, previous)
- Volume control with slider and voice commands
- Favorites management
- Play history tracking
- Custom playlist creation and management
- Sleep timer functionality
- Smart song recommendations
- Beautiful responsive web UI with gradient design
- Quick action buttons for common moods and genres
- 20+ REST API endpoints
- Unit tests for service layer (25+ tests)
- Docker support with multi-stage build
- Docker Compose configuration
- Complete API documentation
- Development guide
- Global exception handling
- Health check endpoints

### Changed
- Redesigned web interface with modern UI
- Enhanced MusicService with full feature set
- Improved QwenService with local fallback parsing
- Updated README with comprehensive documentation

### Fixed
- Volume bounds checking (0-100)
- Proper error handling for API requests
- CORS configuration for cross-origin requests

## [0.2.0] - 2026-03-13

### Added
- Basic music playback functionality
- Qwen AI integration skeleton
- Simple web interface
- REST API endpoints for play/pause/next

### Fixed
- Critical bugs in Spring annotations
- Controller mappings
- HTML/CSS syntax errors
- pom.xml dependencies

## [0.1.0] - 2026-03-11

### Added
- Initial project setup

- Basic project documentation
